PicZTream

$ pzt

终端里的全键盘照片筛选与色彩处理工具。零延迟选片，高性能本地色彩流水线。

macOS Apple Silicon Homebrew C++20 · 零网络依赖（核心）
安装 GitHub
仅支持 macOS / Apple Silicon（M 系列芯片）。 依赖 CoreGraphics/ImageIO 系统框架， 色彩流水线针对 arm64 调优；Intel Mac 与其它平台暂不支持。

它能做什么

零延迟选片

h/l 翻页、space 打标签、x 标废片，配合预取缓存做到按键即出图（Ghostty + Kitty 图像协议）。

本地色彩流水线

一组「城市+年份」recipe 预设，可在预设上自建 version 微调（对比度/饱和度/白平衡/颗粒…），实时预览。

AI 辅助（可选）

看图点评（文字 + 硬伤标记，不打分）、去重/选片用 AI 锦标赛式两两比较。本地 Ollama（免配额）或云端 Claude/Gemini，不占核心执行路径。

Telegram agent

发照片 → 一句话说想怎么弄 → 收成品。收图、按需去重、AI 比较选片、套风格、交付的半自动闭环。

安装 CLI

$ brew tap wangliyangleon/pzt
$ brew trust wangliyangleon/pzt   # Homebrew 6 第三方 tap 一次性信任门
$ brew install pzt
$ pzt --version

brew 会一并装好原生依赖（sqlite / libraw / libomp / nlohmann-json）。

快速上手

$ pzt new trip ~/Photos/trip        # 建项目，指向照片目录
$ pzt open trip                     # 进入全键盘选片
$ pzt export trip 精选 ~/Photos/out  # 按标签导出
键作用
h / l上一张 / 下一张
j / k跳到下一张 / 上一张未打标签的图
space给当前图打标签（分组）
x标记为废片
e / g / r导出当前图 / 筛选 / 风格
q退出

数据存在本地 SQLite：~/.config/pzt/pzt.db，照片本身不搬动。

安装 Telegram agent

半自动选片-交付 bot，单用户自托管。装 pzt-agent 会顺带装 pzt。

$ brew tap wangliyangleon/pzt
$ brew trust wangliyangleon/pzt
$ brew install pzt-agent           # 顺带装 pzt

# 本地 AI 模型（默认 provider=local）
$ brew install --cask ollama
$ ollama pull gemma4:e2b           # 默认模型，可换成别的 Ollama 模型（见「配置」）

# 配置（@BotFather 建 bot 拿 token，取自己的 chat_id）并启动
$ cp "$(brew --prefix)/share/pzt-agent/pzt-agent.env.example" ~/.pzt-agent.env
$ $EDITOR ~/.pzt-agent.env           # 填 TELEGRAM_BOT_TOKEN / TELEGRAM_CHAT_ID
$ set -a; source ~/.pzt-agent.env; set +a
$ pzt-agent                          # 长驻：tmux new -s pzt 'pzt-agent'

前置：一个 Telegram bot（token + 你的 chat_id）、一个 AI 模型（本地 Ollama 或云端 key）。 详见 agent/README.md。

配置

pzt CLI — ~/.config/pzt/config.json

一个 JSON 文件（不存在就全用默认；逐字段独立容错，某个字段写错只回退那一个）。默认走本地 Ollama，换云端就把 ai_provider 设成 gemini/claude。例：

{
  // 换云端 provider / 换本地视觉模型
  "ai_provider": "gemini",
  "ollama_model": "qwen2.5vl:3b"
}
字段默认说明
ai_providerlocal视觉评估 provider：local（本地 Ollama）/ gemini / claude
ollama_modelgemma4:e2b本地视觉模型名（看图点评/去重-选片比较用）；换模型改这里
ollama_base_urllocalhost:11434本地 Ollama 服务地址
curate_time_window_seconds
curate_hash_threshold		20
10		选片分簇的时间窗（秒）与哈希阈值
dedup_time_window_seconds
dedup_hash_threshold		10
5		近似重复分簇的时间窗（秒）与哈希阈值
auto_ai_rejectfalseAI 点评判定有硬伤（unusable）就自动打「废片」标签
eval_reject / dedup_reject
export_reject / export_dup		false批量操作的默认排除策略开关（改成 true 即「不排除」）
lang跟随系统界面语言：zh / en
ui_width_ratio0.7图片显示宽度占终端的比例
prefetch_window3预取缓存窗口（当前图前后各 N 张）

agent — 环境变量

放进 ~/.pzt-agent.env（从 pzt-agent.env.example 拷）。

变量默认说明
TELEGRAM_BOT_TOKEN必填@BotFather 的 bot token
TELEGRAM_CHAT_ID必填只接受这个 chat 的消息（单用户自托管）
PZT_AGENT_META_PROVIDERlocal语言推理 provider：local / gemini / claude
PZT_AGENT_OLLAMA_MODELgemma4:e2bagent 语言推理的本地模型名；换模型改这里
PZT_BINPATH 上的 pzt指定 pzt 二进制；brew 装的一般不用设

命令行参数（状态目录、各种播报/提醒间隔）用 pzt-agent --help 查，状态默认落在 ~/.pzt-agent。

两个本地模型是分开的：CLI 的视觉模型（看图点评/去重-选片比较，config.json 的 ollama_model） 与 agent 的语言模型（意图/风格匹配，PZT_AGENT_OLLAMA_MODEL）各自独立、都可换，默认都是 gemma4:e2b。

升级

$ brew update && brew upgrade pzt pzt-agent
